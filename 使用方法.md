# DeepSeek Free API Proxy

将 DeepSeek 网页端免费对话反代为 OpenAI 兼容接口，支持工具调用。

## 一键部署

```bash
# 下载部署包后
tar -xzf ds-free-api.tar.gz
cd ds-free-api
bash deploy.sh          # 前台启动
bash deploy.sh --bg     # 后台启动
```

或一行安装（如果部署包在可访问的 URL）：
```bash
curl -sL <url>/deploy.sh | bash
```

## 手动部署

```bash
pip install fastapi uvicorn curl-cffi python-dotenv
cd ~/ds-free-api
python3 proxy.py
```

## 配置账号

启动后打开 http://localhost:8000/admin，两种方式：

### 方式一：手机号/邮箱登录（推荐）
1. 选择「手机号登录」或「邮箱登录」
2. 输入账号密码
3. 点击「登录」

### 方式二：浏览器 cURL
1. 打开 chat.deepseek.com 并登录
2. 随便发一条消息
3. F12 → Network → 找到 completion 请求
4. 右键 → Copy as cURL (bash)
5. 粘贴到管理页面 → 保存

## 客户端配置

| 配置项 | 值 |
|--------|-----|
| API 地址 | `http://localhost:8000/v1` |
| API Key | 任意填写 |
| 模型 | `deepseek-v4-flash`（快速）/ `deepseek-v4-pro`（深度思考） |

支持任何 OpenAI 兼容客户端：Chatbox、RikkaHub、Hermes 等。

## 工具调用

代理内置 agentic loop，即使客户端不支持 function calling 也能使用工具：

- **获取时间**：问"现在几点"→ 自动调用 get_current_time
- **执行命令**：让模型执行 shell 命令 → 自动调用 run_terminal_command
- **搜索**：搜索功能暂不可用

如果客户端支持 tools 参数（如 Hermes），会正常返回 tool_calls 格式。

## API 端点

| 端点 | 说明 |
|------|------|
| `GET /health` | 健康检查 |
| `GET /v1/models` | 模型列表 |
| `POST /v1/chat/completions` | 对话（OpenAI 兼容） |
| `GET /admin` | 管理页面 |

## 模型说明

| 模型名 | 功能 | thinking |
|--------|------|----------|
| deepseek-v4-flash / deepseek-default | 普通对话 | 否 |
| deepseek-v4-pro / deepseek-reasoner | 深度思考 | 是 |
| deepseek-search | 联网搜索 | 否 |

## 管理命令

```bash
# 启动（前台）
cd ~/ds-free-api && python3 proxy.py

# 启动（后台）
cd ~/ds-free-api && python3 proxy.py &
# 或
bash deploy.sh --bg

# 停止
pkill -f "python.*proxy.py"

# 查看日志（后台模式）
tail -f /tmp/dsapi.log
```

## 常见问题

### Token 过期
代理会自动重新登录。如果自动登录失败，到管理页面重新登录。

### 工具不生效
确认客户端发送了 `tools` 参数。如果不支持（如 RikkaHub），代理会自动注入默认工具。

### 401 错误
Token 过期，到管理页面重新登录。

### PoW 失败
需要 Node.js。确认 `node --version` 有输出。
